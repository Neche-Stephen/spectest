    API testing without the pain

    Write tests as fast as you think. Spectest transforms API testing from a chore into a superpower with declarative syntax and lightning-fast execution.

    Why Developers Choose Spectest

    Ridiculously Fast
    Execute hundreds of tests in seconds with parallel execution and smart caching.
     Truly Declarative
    Write tests in plain JSON/YAML. No boilerplate, just describe what you expect.
     Focused Workflow
    Built for API testing. Smart defaults and intuitive syntax keep you focused.
     CI/CD Native
    Snapshot testing, environment variables, and detailed reporting for seamless integration.
     Framework Agnostic
    Works with Jest, Vitest, or standalone. Fits into any existing test suite.
     Visual Clarity
    Clear output with diff views and smart error messages saves debugging time.

    See It In Action

    Here’s how simple API testing becomes with Spectest:

    Before: Traditional API Testing 

    describe('User API', () => {
  it('should create user and return 201', async () => {
    const response = await fetch('/api/users', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ name: 'John', email: 'john@example.com' })
    });
    
    expect(response.status).toBe(201);
    const user = await response.json();
    expect(user).toHaveProperty('id');
    expect(user.name).toBe('John');
    expect(user.email).toBe('john@example.com');
  });
});

    After: Spectest Declarative Testing 

    name: "Create User"
endpoint: "/api/users"
method: POST
body:
  name: "John"
  email: "john@example.com"
expect:
  status: 201
  body:
    id: "@type:number"
    name: "John"
    email: "john@example.com"
    That’s it! No setup, no boilerplate, no ceremony. Just describe what you want to test and Spectest handles the execution, validation, and reporting.

    Quick Start Guide

    1. Install
    Get up and running in seconds with our CLI tool
     2. Create Test Suite
    Write your first test in simple YAML format
     3. Run Tests
    Watch your tests execute with beautiful, detailed output

    Install Spectest 

    npm install -g spectest

    Create Your First Test 

    # api-tests.yaml
baseURL: "https://api.example.com"
tests:
  - name: "Health Check"
    endpoint: "/health"
    expect:
      status: 200

    Run Your Tests 

    spectest run api-tests.yaml

    Powerful Features for Every Use Case

    Smart Test Dependencies 

    tests:
  - name: "Login"
    operationId: "auth"
    endpoint: "/auth/login"
    body:
      username: "admin"
      password: "secret"
    
  - name: "Get Profile"
    dependsOn: ["auth"]
    endpoint: "/user/profile"
    headers:
      Authorization: "Bearer {{auth.response.token}}"

    Environment Variables & Configuration 

    // spectest.config.js
module.exports = {
  baseURL: process.env.API_URL || 'http://localhost:3000',
  timeout: 5000,
  retries: 2,
  parallel: true
};

    Snapshot Testing for Regression Prevention 

    # Capture current API responses
spectest run --snapshot

# Later, detect any changes
spectest run --check-snapshots

    Integration That Just Works

    With Jest 

    import { runSpectest } from 'spectest';

describe('API Integration', () => {
  it('should pass all API tests', async () => {
    const results = await runSpectest('api-tests.yaml');
    expect(results.failed).toBe(0);
  });
});

    With Vitest 

    import { describe, it, expect } from 'vitest';
import { runSpectest } from 'spectest';

describe('API Tests', () => {
  it('validates all endpoints', async () => {
    const results = await runSpectest('./tests/*.spec.yaml');
    expect(results.passed).toBeGreaterThan(0);
  });
});
